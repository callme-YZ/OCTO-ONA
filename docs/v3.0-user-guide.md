# OCTO-ONA v3.0 用户指南

## 概述

OCTO-ONA v3.0 引入了**对话式组织网络分析（Conversational ONA）**能力，让您可以用自然语言查询和分析组织网络数据。

### 核心特性

✨ **自然语言查询** — 用日常对话方式提问，无需学习复杂命令

🔐 **细粒度权限控制** — 只能查询自己和自己拥有的 Bot 数据

🤖 **智能响应** — 自动生成自然语言摘要和可视化报告

💬 **多轮对话** — 支持代词消解和上下文继承

📊 **5 大场景** — 个人洞察、团队概览、Bot 监控、趋势追踪、定期报告

## 5 种查询类型

### 1️⃣ 指标查询（Metrics Query）

查询个人或 Bot 的关键指标。

**示例问题：**
```
我的 Hub Score 是多少？
我的活跃度如何？
我本周的消息数
Bot1 的 Hub Score
```

**响应示例：**
```
📊 您的 Hub Score 是 2.5（中等）

这表示您在团队中处于较为核心的位置。

💡 建议：
• 主动回复他人消息，增加互动
• 分享有价值的信息和见解
```

### 2️⃣ 网络分析（Network Query）

查看协作关系和网络连接。

**示例问题：**
```
我和谁有联系？
我的协作网络
显示我的关系图
Bot1 和谁有互动？
```

**响应示例：**
```
🌐 您的协作网络包含 5 个连接：

• Alice（强连接）
• Bob（中等连接）
• Charlie（弱连接）
...
```

### 3️⃣ 排名查询（Ranking Query）

查看团队排名和 Top N。

**示例问题：**
```
Top 5 最活跃的人是谁？
Hub Score 排名前十
本周谁最活跃？
```

**响应示例：**
```
🏆 活跃度排名（Top 5）

🥇 1. Alice - 活跃度：High
🥈 2. Bob - 活跃度：Medium
🥉 3. Charlie - 活跃度：Medium
4. David - 活跃度：Low
5. Eve - 活跃度：Low
```

### 4️⃣ 趋势分析（Trend Analysis）

追踪时间序列变化。

**示例问题：**
```
我的活跃度趋势如何？
本周的趋势
最近一个月的变化
Hub Score 走势
```

**响应示例：**
```
📈 您的活跃度趋势

本周相比上周：+8% ↗️（上升）

• 2026-03-01: 100
• 2026-03-08: 120 (+20%)
• 2026-03-15: 150 (+25%)

趋势良好，保持当前节奏！
```

### 5️⃣ 报告生成（Report Generation）

生成详细的分析报告（HTML 格式）。

**示例问题：**
```
给我一份本周报告
生成个人分析报告
导出 HTML 报告
```

**响应：**
- 文本摘要
- HTML 附件（可在浏览器打开）

## 多轮对话

### 代词消解

OCTO-ONA 能理解对话中的代词（他/她/它）。

**示例：**
```
用户: Alice 的 Hub Score 是多少？
Bot: 😔 很抱歉，您无权查询用户 Alice 的数据...

用户: 她的活跃度呢？
Bot: [自动识别 "她" = Alice，继续权限检查]
```

### 时间范围继承

在同一对话中，时间范围会自动继承。

**示例：**
```
用户: 本周的趋势如何？
Bot: 📈 本周趋势...

用户: 我的活跃度呢？
Bot: [自动继承 "本周" 时间范围]
```

## 权限控制

### 可查询的数据

您**只能**查询：
1. **您自己的数据**（"我的..."）
2. **您拥有的 Bot 数据**

### 权限拒绝示例

```
用户: 小A 的 Hub Score 是多少？

Bot: 😔 很抱歉，您无权查询用户 小A 的数据。

您可以查询：
• 您自己的数据（"我的..."）
• 您拥有的 Bot：
  - Bot1
  - Bot2

💡 试试这些问题：
• "我的 Hub Score 是多少？"
• "我的活跃度如何？"
• "Bot1 的数据"
```

## 错误处理

### 常见错误类型

#### 1. 权限拒绝
```
😔 很抱歉，您无权查询该数据。
[包含允许查询的目标列表和建议问题]
```

#### 2. API 错误
```
😔 很抱歉，系统暂时无法处理您的请求。

可能的原因：
• 分析服务正在维护
• 网络连接暂时中断
• 数据量过大，处理超时

您可以：
• 稍后重试（1-2 分钟后）
• 缩小查询范围（如限制时间范围）
```

#### 3. 数据不足
```
😔 很抱歉，没有找到相关数据。

可能的原因：
• 该用户/Bot 还没有活动记录
• 数据还在统计中（通常需要 1-2 小时）

💡 试试这些：
• 检查用户名/Bot 名是否正确
• 查询您自己的数据确认系统运行正常
```

#### 4. 解析错误
```
😔 很抱歉，我没太理解您的问题。

💡 试试这些问题：
• "我的 Hub Score 是多少？"
• "本周的活跃度如何？"
• "Top 5 最活跃的人是谁？"

输入 /help 查看更多示例。
```

## 常见问题（FAQ）

### Q1: 我可以查询其他人的数据吗？

**A:** 不可以。出于隐私保护，您只能查询：
- 您自己的数据
- 您拥有的 Bot 数据

### Q2: 时间范围如何指定？

**A:** 支持以下时间表达：
- "本周"、"上周"
- "本月"、"上月"
- "最近 7 天"、"最近一个月"
- 自定义日期范围（2026-03-01 到 2026-03-15）

### Q3: 如何导出报告？

**A:** 使用报告生成查询：
```
给我一份本周报告
生成个人分析报告
```

系统会返回 HTML 附件，可在浏览器打开或打印。

### Q4: 数据多久更新一次？

**A:** 数据每小时同步一次，通常会有 1-2 小时延迟。

### Q5: 支持哪些语言？

**A:** 当前支持中文（简体）和英文。报告模板支持双语切换。
